1) Проверка версии гит, который установлен на компьютер (редко)
git --version

2) инициализация пространства git (иногда)
git init

3) проверить статут залитых и не залитых файлов (всегда)
git status

4) зафексировать изменения, еще не закоммитить(сохранить), подготовить файлы к коммиту(сохранению) (всегда)
git add CW_27.py  - только 1 конкретный файл  ( git add --all - все файлы)

5) Создать коммит (сохронить текущее стостояние файла/файлов) (всегда)
git commit -m "add CW_27.py"

6) указываем в какой ветки работает (пока работаем с одной основной веткой) (часто)
git branch -M main

7) присоедениться к репозиторию на гит хаб (редко, мы 1 раз подключились к репозиторию)
git remote add origin ...адрес...

8) залить файлы на гит хаб (всегда)
git push -u origin main

*************************
4 постоянные команды:
git status
git add --all
git commit -m "..."
git push -u origin main
